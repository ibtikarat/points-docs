Points provides official backend SDKs to help you call the same API documented in the API Reference with less boilerplate.

## When to use an SDK

Use an SDK when you want to:

- initialize a typed client once and reuse it across your backend
- reduce manual request signing and header handling
- speed up integration in Laravel, PHP, Node.js, Express, or NestJS apps
- use built-in helpers for webhooks and order actions

Use direct REST calls if your stack does not use PHP or Node.js, or if you prefer to work directly with HTTP.

## What stays the same

Both SDKs wrap the same Points API concepts:

- checkout creation
- earning and redemption order flows
- order lifecycle actions such as authorize, capture, and complete
- webhook parsing and verification helpers

## Authentication model

In both SDKs:

- the private key is used for authenticated backend operations
- the public key is used when a checkout-related endpoint needs it
- webhook verification uses your webhook secret

<Note>
The SDKs are for backend usage. Do not expose your private key in frontend code.
</Note>
